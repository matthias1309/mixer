# Recipe Manager v1.0.0

**Release Date**: May 16, 2026

---

## Overview

Recipe Manager v1.0.0 is the first production-ready release of a multi-user recipe management application with menstrual cycle-aware nutrition recommendations.

## Features

### Core Recipe Management
- ✅ Create, read, update, delete recipes
- ✅ Recipe ingredients with quantities and units
- ✅ Recipe descriptions and cooking instructions
- ✅ Full-text recipe search
- ✅ Recipe ownership and permissions

### Nutrition & Ingredients
- ✅ 29 pre-seeded nutrition ingredients with comprehensive nutrient data
- ✅ Ingredient master database with nutritional profiles (kcal, protein, fat, carbs, fiber, and 10+ micronutrients)
- ✅ Automatic nutrient calculation for recipes based on ingredients
- ✅ Display of macro and micronutrients per recipe

### Menstrual Cycle Integration
- ✅ User cycle tracking with customizable cycle length (21-35 days)
- ✅ Four cycle phases: menstruation, follicular, ovulation, luteal
- ✅ Phase-aware recipe scoring based on nutritional needs
- ✅ Automatic phase calculation from last menstruation date
- ✅ Phase-based recipe filtering and recommendations

### User Management
- ✅ User registration and login
- ✅ JWT-based authentication with secure password hashing
- ✅ Session token refresh mechanism
- ✅ User profile management

### Technical Architecture
- ✅ Next.js 15 full-stack application
- ✅ TypeScript with strict type checking
- ✅ PostgreSQL for production (with SQLite fallback for local development)
- ✅ Dual-database support for seamless local→production workflow
- ✅ Docker containerization for Raspberry Pi deployment
- ✅ API Routes with middleware for authentication and database initialization
- ✅ Responsive UI with Tailwind CSS

## Deployment

### Production
- **Target**: Raspberry Pi with Docker
- **Database**: PostgreSQL 16-alpine
- **Port**: 3001
- **Location**: `/opt/containers/apps/mixer`
- **Access**: SSH via `ssh dockerhome`

### Local Development
```bash
# One-time setup
npm run db:setup        # Start PostgreSQL + run migrations
npm run db:seed         # Seed 29 example ingredients

# Daily development
npm run dev             # Start Next.js dev server

# Cleanup
npm run db:stop        # Stop PostgreSQL container
```

## Key Improvements from Pre-Release

### PostgreSQL Compatibility
- Converted all synchronous SQLite-only database methods to async
- Implemented dual-database support with `isPostgres()` branching
- Fixed PostgreSQL-specific SQL issues (GROUP BY strictness, NULL type inference, DECIMAL→string conversion)
- Added proper type conversion for numeric values from PostgreSQL

### Stability & Testing
- Added comprehensive async/await error handling in all API routes
- Improved database initialization with proper health checks
- Added seed data for rapid local development
- Tested all CRUD operations on PostgreSQL production environment

### Developer Experience
- Created local development environment setup script (`npm run db:setup`)
- Separated development database (PostgreSQL Docker) from production
- Added seed data command for quick testing (`npm run db:seed`)
- Improved `.env` documentation with connection strings

## Known Limitations & Future Work

### Not Included in v1.0
- Recipe import from external APIs
- Photo-based ingredient recognition
- Advanced meal planning
- Shopping list generation
- Recipe rating and reviews
- User profiles and settings
- Nutrient-based ingredient filtering (beyond cycle phase)

### Technical Debt
- Migration system lacks version tracking (all migrations re-run on startup)
- Two migration directories exist (`src/lib/db/migrations/` and `src/db/migrations/`) — only the former is used
- Some ESLint warnings disabled during build (see MAINT-001 ticket)
- Phase nutrient targeting is basic (could be optimized per phase)

## Installation & Running

```bash

git clone <repo>
cd mixer
npm install

# Setup database
npm run db:setup

# Seed example data
npm run db:seed

# Open browser
# Navigate to http://localhost:3000
```

### Production Deployment
```bash
# Via SSH on Raspberry Pi
./scripts/deploy-pi.sh \
  --db-password "secure-password" \
  --jwt-secret "min-32-char-secret"

# Clean reset (delete all data)
ssh dockerhome 'cd /opt/containers/apps/mixer && docker compose -f docker-compose.production.yml down -v'
./scripts/deploy-pi.sh --db-password "..." --jwt-secret "..."
```

## Database Schema

### Core Tables
- `users` — registered users with email and password hash
- `recipes` — user-created recipes with name, description, instructions
- `ingredients` — recipe ingredients (name, quantity, unit) linked to recipes
- `nutrition_ingredients` — master ingredient database with nutritional data
- `recipe_nutrients` — pre-calculated total and per-portion nutrients for recipes

### Cycle Tracking
- `user_cycles` — user menstrual cycle information (start date, cycle length)

## Security

- ✅ Password hashing with bcryptjs (salt rounds: 10)
- ✅ JWT tokens with configurable expiration (default: 24h)
- ✅ HTTP-only session cookies
- ✅ CORS configuration per environment
- ✅ Input validation on all API endpoints
