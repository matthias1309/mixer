# REQ-005 — UX/UI Verbesserungen Phase 1

**Status:** Implemented
**Created:** 2026-06-11
**Traced by:** ARCH-005, TEST-005

## User Story

As a home cook using the Recipe Manager,
I want a clearer and more intuitive user interface,
in order to find recipes faster and use them more efficiently while cooking.

## Acceptance Criteria

### AC-005-01: Recipe detail page — logical content order

```gherkin
Given a user opens a recipe detail page
When the page finishes loading
Then the content appears in this order: title, description, servings control, ingredients, instructions, nutrients
And the nutrients section is collapsed by default
And the user can expand the nutrients section with a single click
```

### AC-005-02: Dashboard — clear primary CTA above recipe list

```gherkin
Given a logged-in user opens the dashboard
When the page finishes loading
Then the primary call-to-action "Rezept erstellen" is visible above the recipe list without scrolling
And secondary actions (photo upload, cycle, ingredients) are visually subordinate to the primary action
And all action buttons share a consistent styling hierarchy (one primary, rest secondary/outlined)
```

### AC-005-03: Phase filter — chips instead of dropdown

```gherkin
Given a user is on the dashboard
When they look at the phase filter
Then the four cycle phases are displayed as clickable chip buttons instead of a dropdown
And the currently active phase chip is visually highlighted
And selecting a chip immediately updates the recipe list
```

### AC-005-04: Wake lock button — icon instead of text label

```gherkin
Given any user views the navigation bar
When the wake lock button is displayed
Then it shows a recognisable icon (e.g. sun/screen) instead of the text "Bildschirm: AN/AUS"
And a tooltip on hover shows the full description
And the active/inactive state is still visually distinguishable
```

## Notes

- This story covers UI-only changes — no new API endpoints or data model changes required.
- Wake lock is only shown when the browser supports it (`isSupported === true`).
- The nutrients collapse state does not need to persist across page loads.
- Secondary actions on the dashboard should remain accessible; only visual hierarchy changes, not removal.
- Phase 2 UX improvements (recipe photos, mobile nav, skeleton loaders, delete modal) are out of scope here and tracked separately.
