# REQ-004 — Public Read Access

**Version**: 1.0
**Date**: 2026-05-30
**Status**: Implemented
**Epic**: Phase 2 — UX Improvements
**Priority**: Should Have
**Owner**: Matthias Bender
**Traced by**: ARCH-004, TEST-004

---

## 1. Overview

**Description**: The application should be accessible without authentication for read-only use. Users must only log in when they want to create, edit, or delete content.

**Business Value**:
- Lower barrier to entry — visitors can browse recipes immediately
- Better user experience on shared/household devices (Raspberry Pi home server)
- Public shareability of recipes without forcing recipients to register

**Success Criteria**:
- Unauthenticated users can view the recipe list (dashboard)
- Unauthenticated users can view recipe details
- Unauthenticated users can browse the ingredient master list
- Unauthenticated users cannot create, edit, or delete any content
- Authenticated users retain full access to all write operations

---

## 2. Functional Requirements

### FR-401: Public Recipe Browsing

- **Description**: The recipe list and recipe detail pages are accessible without login
- **Acceptance Criteria**:
  - AC-004-01: Visiting `/dashboard` without a session cookie shows the recipe list
  - AC-004-02: Visiting `/recipes/[id]` without a session cookie shows the full recipe detail
  - AC-004-03: The ingredient filter and phase filter work without login
  - AC-004-04: No redirect to `/login` occurs when browsing recipes unauthenticated

### FR-402: Public Ingredient Browsing

- **Description**: The ingredient master list is accessible without login
- **Acceptance Criteria**:
  - AC-004-05: Visiting `/ingredients` without a session cookie shows the ingredient list
  - AC-004-06: Search within the ingredient list works without login
  - AC-004-07: Add/Edit/Delete controls are not shown to unauthenticated users

### FR-403: Protected Write Operations

- **Description**: Creating, editing, and deleting content still requires authentication
- **Acceptance Criteria**:
  - AC-004-08: Visiting `/recipes/new` without login redirects to `/login`
  - AC-004-09: Visiting `/recipes/[id]/edit` without login redirects to `/login`
  - AC-004-10: Visiting `/ingredients/new` without login redirects to `/login`
  - AC-004-11: API endpoints `POST/PUT/DELETE` return `401 Unauthorized` without a valid session

### FR-404: Homepage Redirect

- **Description**: The root URL `/` redirects directly to the recipe list
- **Acceptance Criteria**:
  - AC-004-12: Visiting `/` redirects to `/dashboard` for both authenticated and unauthenticated users

### FR-405: Navigation for Unauthenticated Users

- **Description**: The navigation shows public links to all users
- **Acceptance Criteria**:
  - AC-004-13: "Rezepte" and "Zutaten" links are visible without login
  - AC-004-14: "Anmelden" and "Registrieren" links are shown when not logged in
  - AC-004-15: Auth-specific links (Zyklus, Abmelden) are hidden when not logged in

---

## 3. Non-Functional Requirements

- **Security**: Write-protecting API endpoints via server-side auth check is mandatory — client-side hiding of buttons is UI-only and not a security boundary
- **Performance**: No additional API calls for unauthenticated page loads
