# REQ-007 — UX/UI Verbesserungen Phase 2b: Skeleton Loader & Zutatenfilter-Suche

**Status:** Implemented
**Created:** 2026-06-11
**Traced by:** ARCH-007, TEST-007

## User Story

As a user of the Recipe Manager,
I want smooth loading feedback and a searchable ingredient filter,
in order to understand when content is loading and find ingredients quickly in long lists.

## Acceptance Criteria

### AC-007-01: Skeleton loading state for recipe list

```gherkin
Given a user navigates to the dashboard
When the recipe list is being fetched
Then placeholder skeleton cards are shown instead of a plain text spinner
And the skeletons match the approximate shape of real recipe cards
```

### AC-007-02: Searchable ingredient filter

```gherkin
Given a user is on the dashboard with the ingredient filter visible
When they type in the filter search field
Then only ingredients whose names contain the search text are shown in the list
And clearing the search field restores the full ingredient list
```

## Notes

- Skeleton cards: 3 grey placeholder blocks, no animation library required (CSS pulse via Tailwind `animate-pulse`).
- Ingredient search is client-side only — no new API calls.
- The existing "Filter löschen" button and selected-ingredient chips are unaffected by the search.
- Matching is case-insensitive.
