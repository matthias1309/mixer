# REQ-018 — Recipe Star Ratings

**Version**: 1.0
**Date**: 2026-06-25
**Status**: draft
**Epic**: Phase 3 — UX Overhaul (REWE-Style Redesign)
**Priority**: P3 (Feature)
**Owner**: Matthias Bender
**Traced by**: ARCH-018, TEST-018

> Part 4 of 4 of the REWE-style redesign. This story adds a 1–5 **star rating**
> per user per recipe (REWE shows e.g. "4.7/5"), the aggregate on cards, a
> "Bewertung mindestens" filter, and a `sort=rating` ordering. It closes the
> data gap that REQ-017's rating filter/sort depends on.

---

## 1. Overview

**Description**: Let an authenticated user rate a recipe 1–5 stars (one rating
per user per recipe, changeable). The card and detail view show the average and
the count; the filter panel gains a minimum-rating control; recipes can be
sorted by average rating.

**Business Value**:
- Social proof and quality signal, exactly like the REWE portal
- Completes the REWE filter set (`minRating`, `sort=rating`)

---

## 2. Functional Requirements

### FR-1801: Schema migration

- **Acceptance Criteria**:
  - AC-018-01: Migration `012` creates `recipe_ratings`
    (`user_id`, `recipe_id`, `stars` 1–5, `created_at`/`updated_at`) with FKs to
    users and recipes (ON DELETE CASCADE) and a uniqueness constraint on
    (`user_id`, `recipe_id`).

### FR-1802: Submit / change a rating

- **Acceptance Criteria**:
  - AC-018-02: An authenticated `POST`/`PUT` to a rating endpoint with
    `stars ∈ 1..5` records (or updates) the caller's rating for that recipe.
  - AC-018-03: `stars` outside 1..5 is rejected with HTTP 400.
  - AC-018-04: An unauthenticated rating attempt returns HTTP 401.
  - AC-018-05: A second rating by the same user on the same recipe **updates**
    the existing row rather than creating a duplicate.

### FR-1803: Aggregate exposure

- **Acceptance Criteria**:
  - AC-018-06: Recipe list/detail responses include `ratingAverage`
    (rounded to one decimal) and `ratingCount`; a recipe with no ratings reports
    average `null` / count `0`.
  - AC-018-07: The card shows the average and count (e.g. "4.7 ★ (12)") when
    `ratingCount > 0`, and a neutral "noch keine Bewertung" state otherwise.

### FR-1804: Rating widget

- **Acceptance Criteria**:
  - AC-018-08: The recipe detail page renders an interactive 1–5 star control
    that reflects the user's current rating and submits changes.

### FR-1805: Filter & sort by rating

- **Acceptance Criteria**:
  - AC-018-09: `GET /api/recipes?minRating=N` returns only recipes whose average
    rating is ≥ N (recipes with no ratings are excluded when `minRating > 0`).
  - AC-018-10: `GET /api/recipes?sort=rating` orders by average rating
    descending; the filter panel exposes the minimum-rating control.

### FR-1806: Quality gates

- **Acceptance Criteria**:
  - AC-018-11: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass.

## 3. Non-Functional Requirements

- **SQLite-only** (ADR-008); aggregates computed in SQL (`AVG`, `COUNT`).
- **Migration review**: migration `012` reviewed before production (CLAUDE.md).
- **Integrity**: one rating per (user, recipe) enforced at the DB level, not only
  in app code.
- **Composition**: `minRating` / `sort=rating` plug into the REQ-017 engine and
  compose with phase, ingredient, and metadata filters.

## 4. Out of Scope

- Written reviews / comments (ratings only)
- Anti-abuse / rate-limiting beyond the one-row-per-user constraint
