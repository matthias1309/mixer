## 1. Overview

**Description**: Each recipe is scored 0–100 for a given menstrual cycle phase,
based on how well its aggregated nutrients match that phase's priorities. The
score is returned alongside recipes on the main listing endpoint, so the
dashboard can rank/communicate which recipes suit the user's current phase
(REQ-010).

**Business Value**:
- Turns the nutrition data (REQ-012) and cycle phase (REQ-010) into an
  actionable, per-recipe signal
- Helps the user pick recipes that fit their current phase

---

## 2. Functional Requirements

### FR-1101: Phase-weighted recipe score

- **Description**: A recipe's aggregated nutrients are scored against
  phase-specific nutrient weights, normalised to 0–100.
- **Acceptance Criteria**:
  - AC-011-01: The score for any recipe/phase is between 0 and 100 inclusive
  - AC-011-02: An unknown phase yields a score of 0
  - AC-011-03: A recipe with no nutrient data yields a score of 0

### FR-1102: Phase-specific prioritisation

- **Description**: Each phase weights different nutrients (e.g. Menstruation →
  iron; Luteal → magnesium; Follicular → B-vitamins).
- **Acceptance Criteria**:
  - AC-011-04: An iron-heavy recipe scores higher for **Menstruation** than for
    **Ovulation**
  - AC-011-05: A magnesium-heavy recipe scores highest for **Luteal**
  - AC-011-06: A single extreme nutrient cannot dominate — normalised nutrient
    contributions are capped at 1.0

### FR-1103: Score surfaced on the recipe listing

- **Description**: `GET /api/recipes?phase=<phase>` returns each recipe with a
  computed `score` (or `null` when no master-ingredient matched).
- **Acceptance Criteria**:
  - AC-011-07: `GET /api/recipes?phase=menstruation` returns list items that
    include a `score` field
  - AC-011-08: The same recipe yields different scores for different `phase`
    values
  - AC-011-09: Listing with a phase works for an authenticated user and supports
    pagination

### FR-1104: Quality gates

- **Acceptance Criteria**:
  - AC-011-10: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass

## 3. Non-Functional Requirements

- **Algorithm**: `score = (phaseSpecific × 0.7 + general × 0.3) × 100`, where
  each nutrient is normalised as `min(amount / reference, 1.0)` and weighted by
  the phase. Defined in `src/lib/scoring/phaseScore.ts`.
- **Computation**: Scores are computed at request time from nutrients
  aggregated out of `ingredients_master` (no precomputed/cached score column).
- **Phases**: menstruation | follicular | ovulation | luteal
  (`src/lib/constants.ts`).
- **Database**: SQLite-only (ADR-008).
- **Out of scope (designed but not built)**: dedicated `/recommended`,
  `/filtered`, and `/[id]/phase-scores` routes; a `min_score` filter; the
  alternative `src/lib/cycle-recommendations/scorer.ts` and its
  `PHASE_NUTRIENT_TARGETS` are present but not wired into the API.
