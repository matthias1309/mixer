# REQ-016 — Recipe Metadata & REWE Tag Vocabulary

**Version**: 1.0
**Date**: 2026-06-25
**Status**: draft
**Epic**: Phase 3 — UX Overhaul (REWE-Style Redesign)
**Priority**: P2 (Feature)
**Owner**: Matthias Bender
**Traced by**: ARCH-016, TEST-016

> Part 2 of 4 of the REWE-style redesign. This story introduces the **data
> foundation** for REWE-style filtering: recipe effort (`difficulty`), total
> time, meal type, and a fixed **tag vocabulary copied 1:1 from REWE**. It adds
> the schema, the constants, the create/edit form fields, and surfaces the new
> data on the card — but **not** the filter engine (REQ-017) or ratings
> (REQ-018).

---

## 1. Overview

**Description**: Extend recipes with the metadata REWE exposes, so the app can
later filter by it. `difficulty` and `meal_type` become columns on `recipes`;
the multi-valued categories (Ernährung, Hauptzutat, Ernährungsform, Backen,
Anlässe) become an n:m `recipe_tags` relation against a fixed, REWE-derived
vocabulary. The recipe form lets users set these; the card shows time, effort,
and tag chips.

**Business Value**:
- Unlocks every REWE-style filter (REQ-017) and richer cards
- A fixed vocabulary keeps tags consistent and filterable (no free-text drift)

## 2. Functional Requirements

### FR-1601: Fixed tag vocabulary (from REWE)

- **Acceptance Criteria**:
  - AC-016-01: A central constant defines the vocabulary, grouped exactly as
    REWE: **Ernährung** (Fleisch, Fisch, Vegetarisch, Vegan), **Hauptzutat**
    (Nudeln/Pasta, Kartoffeln, Reis, Gemüse, Kürbis), **Ernährungsform**
    (Laktosefrei, Low Carb, Glutenfrei, Paleo, Wenig Zucker, Clean Eating),
    **Backen** (Kuchen, Torten, Brot, Muffins, Cupcakes, Plätzchen), **Anlässe**
    (Frühling, Grillen, Picknick, Kindergerichte, Geburtstag, Party, günstig).
  - AC-016-02: `difficulty` is the enum `easy | medium | hard` (display:
    Geringer / Mittlerer / Hoher Aufwand). `meal_type` is one of Vorspeise,
    Hauptspeise, Dessert, Beilagen, Frühstück, Suppen, Auflauf, Snacks,
    Getränke.

### FR-1602: Schema migration

- **Acceptance Criteria**:
  - AC-016-03: Migration `011` adds nullable `difficulty`, `total_time_minutes`
    (INTEGER), and `meal_type` columns to `recipes`.
  - AC-016-04: Migration `011` creates a `recipe_tags` table
    (`recipe_id`, `tag`) with a FK to `recipes` (ON DELETE CASCADE), a uniqueness
    constraint on (`recipe_id`, `tag`), and an index on `tag`.
  - AC-016-05: The migration is additive only — existing recipes remain valid
    with NULL metadata and zero tags.

### FR-1603: Create/edit recipe with metadata

- **Acceptance Criteria**:
  - AC-016-06: `CreateRecipeRequest` / `UpdateRecipeRequest` accept optional
    `difficulty`, `totalTimeMinutes`, `mealType`, and `tags: string[]`.
  - AC-016-07: A submitted `difficulty` / `mealType` / `tag` that is not in the
    fixed vocabulary is rejected with HTTP 400.
  - AC-016-08: `totalTimeMinutes`, when present, must be a positive integer,
    else HTTP 400.
  - AC-016-09: `RecipeForm` provides inputs for effort, time, meal type, and
    multi-select tags; saving persists them and editing pre-fills them.

### FR-1604: Surface metadata on cards & detail

- **Acceptance Criteria**:
  - AC-016-10: When present, the card meta row shows total time (e.g. "35 min")
    and effort (e.g. "Einfach"); absent values are simply omitted.
  - AC-016-11: When a recipe has tags, the card renders them as chips in the
    tag slot introduced by REQ-015.

### FR-1605: Quality gates

- **Acceptance Criteria**:
  - AC-016-12: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass.

## 3. Non-Functional Requirements

- **SQLite-only** (ADR-008); raw SQL via better-sqlite3.
- **Migration review**: migration `011` must be reviewed before running in
  production (CLAUDE.md).
- **Backwards compatible**: the existing `POST /api/recipes` contract gains
  optional fields only; clients that omit them keep working.
- **Vocabulary single source of truth**: the lists live in `src/lib/constants.ts`
  and are reused by validation, the form, and (later) the filter UI.

## 4. Out of Scope

- Filtering/sorting by the new fields (REQ-017)
- Star ratings (REQ-018)
- Auto-deriving tags from ingredients (possible future work)
