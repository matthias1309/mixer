# REQ-006 — UX/UI Verbesserungen Phase 2: Mobile Navigation & Löschen-Modal

**Status:** Implemented
**Created:** 2026-06-11
**Traced by:** ARCH-006, TEST-006

## User Story

As a user of the Recipe Manager on any device,
I want a responsive navigation and a safe delete confirmation flow,
in order to use the app comfortably on mobile and avoid accidental data loss.

## Acceptance Criteria

### AC-006-01: Mobile navigation — hamburger menu

```gherkin
Given a user opens the app on a narrow viewport (< 768px wide)
When they view the navigation bar
Then the full link row is hidden
And a hamburger icon button is visible
And clicking the hamburger opens a menu with all navigation links
And clicking a navigation link closes the menu
And clicking outside the open menu closes it
```

### AC-006-02: Delete confirmation — modal instead of browser dialog

```gherkin
Given a logged-in user is on a recipe detail page they are allowed to delete
When they click the "Löschen" button
Then a modal dialog appears in the page (not a browser confirm() dialog)
And the modal displays the recipe name and a warning message
And clicking "Abbrechen" closes the modal and leaves the recipe untouched
And clicking "Löschen" in the modal deletes the recipe and redirects to the dashboard
```

## Notes

- The hamburger menu must include all links shown in the desktop nav: Rezepte, Zutaten,
  Zyklus (if logged in), user email (display only), Abmelden / Anmelden / Registrieren.
- Wake-lock icon button remains visible on mobile at all times (no hamburger required for it).
- The modal must be keyboard-accessible: focus trap inside the modal, Escape closes it.
- No new shared Modal component is introduced unless a second use case already exists in
  this story — inline implementation is preferred (YAGNI).
- The desktop navigation layout (≥ 768px) must remain unchanged.
