## 1. Overview

**Description**: A user can record the start date of their last menstruation
and their typical cycle length. From this the application derives the current
cycle phase (Menstruation, Follicular, Ovulation, Luteal) and the day-of-cycle,
which feeds the cycle-based recipe scoring (REQ-011).

**Business Value**:
- Provides the personal context that powers phase-aware recipe recommendations
- Lets the user see where they are in their cycle at a glance

## 2. Functional Requirements

### FR-1001: Save / update the cycle profile

- **Description**: A user stores their last menstruation date and cycle length;
  re-submitting updates the existing profile (one profile per user).
- **Acceptance Criteria**:
  - AC-010-01: An authenticated `POST /api/users/cycle` with a valid
    `last_menstruation_date` and `cycle_length_days` (21–35) persists the
    profile and returns HTTP 200
  - AC-010-02: A `cycle_length_days` outside 21–35 is rejected with HTTP 400
  - AC-010-03: A request without a valid session returns HTTP 401

### FR-1002: Calculate the current phase

- **Description**: Day-of-cycle is `(today − lastMenstruationDate) mod
  cycleLength`, mapped to one of four phases.
- **Acceptance Criteria**:
  - AC-010-04: For a 28-day cycle, a day-of-cycle of 14 resolves to the
    **Ovulation** phase
  - AC-010-05: When `lastMenstruationDate` equals today, day-of-cycle is 0 and
    the phase is **Menstruation**
  - AC-010-06: The calculation produces a correct phase across different cycle
    lengths

### FR-1003: Retrieve the cycle and current phase

- **Description**: The stored cycle and the freshly-computed current phase are
  retrievable.
- **Acceptance Criteria**:
  - AC-010-07: `GET /api/users/cycle` returns HTTP 200 with the stored cycle and
    a `current_phase` whose name is one of Menstruation/Follicular/Ovulation/Luteal
  - AC-010-08: For a user with no stored cycle, `GET /api/users/cycle` returns
    HTTP 200 with `success: false` / `data: null`

### FR-1004: Quality gates

- **Acceptance Criteria**:
  - AC-010-09: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass

## 3. Non-Functional Requirements

- **Storage**: One row per user in `user_cycles` (`user_id` UNIQUE), created by
  migration `src/lib/db/migrations/003_create_user_cycles.sql`. SQLite-only
  (see ADR-008).
- **Privacy**: Cycle data is sensitive. Endpoints are owner-scoped via the
  authenticated `userId`; there is no sharing or public read.
- **Phase model**: Four phases with fixed day ranges suitable for a 28-day
  reference cycle, defined in `src/lib/cycle/constants.ts`. Overlaps resolve by
  `PHASE_PRIORITY`.
- **Out of scope**: cycle logging/notes, a `/phases` reference endpoint, and a
  `/phase-on-date` HTTP endpoint (the `calculatePhaseOnDate` helper exists in
  `src/lib/cycle/calculator.ts` but is not exposed as a route).
