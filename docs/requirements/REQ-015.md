# REQ-015 — REWE-Style Layout & Recipe Card Redesign

**Version**: 1.0
**Date**: 2026-06-25
**Status**: draft
**Epic**: Phase 3 — UX Overhaul (REWE-Style Redesign)
**Priority**: P2 (Feature)
**Owner**: Matthias Bender
**Traced by**: ARCH-015, TEST-015

> Part 1 of 4 of the REWE-style redesign (REQ-015…018). This story covers the
> **purely visual / structural** reskin — design tokens, page layout, recipe
> card, image fallback, and numbered pagination — using **only the filters that
> already exist** (cycle phase + ingredients). New filter *data* and the
> filter *engine* are out of scope here and handled by REQ-016/REQ-017.
>
> **The cycle-phase filter is a hard requirement and must remain** as the first,
> visually emphasised filter group (the app's differentiator vs. REWE).

---

## 1. Overview

**Description**: Re-skin the recipe overview (`/dashboard`) to match the visual
language of the REWE recipe portal, translated into the project's own fresh
palette (terracotta primary, sage-green accent, warm off-white surface). The
layout becomes: a search bar on top, a left filter sidebar with collapsible
groups, a denser card grid on the right, and numbered pagination. Recipe cards
become image-first with a tag-chip slot and a meta row (time · effort · rating
placeholders). Recipes without a photo render a deterministic gradient
placeholder instead of an empty card.

**Business Value**:
- Familiar, polished, food-photo-driven browsing experience
- Consistent grid (no ragged cards from missing images)
- Establishes the design-token foundation reused by REQ-016–018

## 2. Functional Requirements

### FR-1501: Design-token foundation

- **Acceptance Criteria**:
  - AC-015-01: A central theme defines the palette as named tokens
    (`brand` ≈ `#C65D3B`, `accent` ≈ `#5B8C5A`, `surface` ≈ `#FAF6F1`,
    `ink` ≈ `#2E2A26`) usable as Tailwind utility classes.
  - AC-015-02: No `bg-blue-600` / `text-blue-600` / `border-blue-600` literals
    remain in `Navigation`, `dashboard/page`, `RecipeList`, `Pagination`, or
    `PhaseFilter`; they reference the new tokens instead.

### FR-1502: Recipe card redesign (image-first)

- **Acceptance Criteria**:
  - AC-015-03: A card renders the recipe image at the top at a fixed aspect
    ratio, then the title, then a meta row.
  - AC-015-04: A card exposes a tag-chip slot (rendered only when tags exist;
    tags themselves arrive in REQ-016, so the slot is empty for now).
  - AC-015-05: The phase score continues to render as a badge on the card.

### FR-1503: Image fallback (gradient placeholder)

- **Description**: A `RecipeImage` component decides between the real photo and
  a generated placeholder.
- **Acceptance Criteria**:
  - AC-015-06: When `imagePath` is present, the real image is shown.
  - AC-015-07: When `imagePath` is absent/null, a gradient placeholder is shown
    at the **same dimensions** as a real image, derived **deterministically from
    the recipe id** (same id → same gradient), with a centred icon and the
    recipe name as overlay text.

### FR-1504: REWE-style page layout

- **Acceptance Criteria**:
  - AC-015-08: The dashboard shows a search input above the results area.
  - AC-015-09: Filters live in a left sidebar as collapsible groups within a
    reusable `FilterPanel`; the **cycle-phase group is first and visually
    emphasised**, the ingredient group second.
  - AC-015-10: A "Filter zurücksetzen" control clears all active filters.
  - AC-015-11: A results counter shows the number of matching recipes.

### FR-1505: Numbered pagination

- **Acceptance Criteria**:
  - AC-015-12: Pagination renders numbered page links (1, 2, 3, … N) with the
    current page marked active, replacing the previous "Vorherige/Nächste"
    buttons (prev/next affordances may remain as arrows).

### FR-1506: No regression & quality gates

- **Acceptance Criteria**:
  - AC-015-13: Existing cycle-phase and ingredient filtering behaviour is
    unchanged (same query params, same results).
  - AC-015-14: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass.

## 3. Non-Functional Requirements

- **No database change** in this story (zero migration risk).
- **Responsive**: the sidebar collapses above the grid on mobile; the grid is
  1 / 2 / 3 columns at sm / md / lg.
- **Accessibility**: filter groups are keyboard-operable; the placeholder image
  carries the recipe name as accessible text.
- **Base path**: all links/fetches keep using `apiUrl()` / `next/link`
  (MAINT-003).

## 4. Out of Scope

- New recipe metadata fields and tag vocabulary (REQ-016)
- Filter engine for the new dimensions (REQ-017)
- Star ratings (REQ-018)
