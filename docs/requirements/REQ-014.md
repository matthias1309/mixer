## 1. Overview

**Description**: A user uploads a photo of a recipe; the app extracts the text
via OCR, parses ingredient lines into `{amount, unit, name}`, fuzzy-matches each
name against the ingredient catalogue (REQ-012) with a confidence score, and
presents the result for review before the user creates the recipe.

**Business Value**:
- Lets users add a recipe from a photo instead of typing every ingredient
- Matching against the catalogue gives the new recipe nutrient data for free

## 2. Functional Requirements

### FR-1401: Upload a photo for OCR

- **Acceptance Criteria**:
  - AC-014-01: An authenticated `POST /api/recipes/ocr` with a JPG/PNG (≤ 5 MB)
    accepts the upload and returns an `uploadId` with a processing status
  - AC-014-02: A non-image or oversized file is rejected with HTTP 400
  - AC-014-03: An unauthenticated upload returns HTTP 401

### FR-1402: Extract text via OCR (German + English)

- **Acceptance Criteria**:
  - AC-014-04: Uploaded image text is extracted via tesseract.js configured for
    `deu` + `eng` and made available to the parser/review step

### FR-1403: Parse ingredient lines

- **Description**: Each text line is split into an optional amount, optional
  unit, and an ingredient name.
- **Acceptance Criteria**:
  - AC-014-05: A line like "200 ml Milch" parses to `{amount: 200, unit: "ml",
    name: "Milch"}`
  - AC-014-06: A line with no amount/unit (e.g. "Salz") parses to a name with
    null amount/unit
  - AC-014-07: A multi-line block parses into one entry per line

### FR-1404: Fuzzy-match to the ingredient catalogue with confidence

- **Acceptance Criteria**:
  - AC-014-08: An exact (case-insensitive) name match returns the catalogue id
    with confidence 1.0 and `matched: true`
  - AC-014-09: A near match (typo/diacritic/case) returns a high-confidence
    fuzzy match
  - AC-014-10: An unmatched name returns no id, confidence 0, `matched: false`

### FR-1405: Poll status & review

- **Acceptance Criteria**:
  - AC-014-11: `GET /api/recipes/ocr/[uploadId]` (owner-only) returns the
    processing status and, when complete, the parsed/matched ingredients for
    review and correction in the UI

### FR-1406: Quality gates

- **Acceptance Criteria**:
  - AC-014-12: `npm run lint`, `npx tsc --noEmit`, and `npm test` all pass

## 3. Non-Functional Requirements

- **Async/polling**: upload returns immediately with an `uploadId`; the client
  polls the status endpoint until `complete`/`error`. Intermediate results are
  held in an in-memory cache (`src/lib/ocr/cache.ts`) with periodic cleanup.
- **Matching**: Levenshtein-based similarity plus an `INGREDIENT_SYNONYMS` map;
  thresholds in `src/lib/ocr/constants.ts`.
- **Validation**: reuses `src/config/upload.ts` (JPG/PNG, 5 MB) — shared with
  REQ-009.
- **Database**: SQLite-only (ADR-008); the matcher reads `ingredients` /
  `ingredients_master`.
- **Recipe creation**: the reviewed ingredients are submitted through the
  existing `POST /api/recipes` contract (not a new endpoint).
