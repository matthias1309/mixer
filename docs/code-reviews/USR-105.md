## Key Decisions

1. **Synchronous API** — `generateToken` is sync (jwt.sign is sync without callback)
2. **Null return on errors** — `verifyToken` and `decodeToken` return `null` on error instead of throwing (safer for auth middleware)
3. **Secret validation early** — `getSecret()` validates JWT_SECRET on every token generation (fail-fast pattern)
4. **Type cast for SignOptions** — `as any` cast for SignOptions is necessary due to @types/jsonwebtoken definition limitations

✅ **Secret Management**

✅ **Token Claims**
