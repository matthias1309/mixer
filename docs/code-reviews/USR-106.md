1. **`src/lib/auth/jwt.ts`** (modified - 7 new LOC)

2. **`src/lib/auth/middleware.ts`** (new - 33 LOC)
   - `authMiddleware(req: NextRequest): NextResponse`
   - Extracts token from cookies via `req.cookies.get('token')`
   - Returns 401 Unauthorized for missing/invalid tokens
   - Returns 403 Forbidden for expired tokens
   - Forwards user info via custom request headers (x-user-id, x-user-email)

## Key Decisions

1. **TokenVerificationResult Type** — Union type over boolean+error pair because it forces explicit error handling
2. **verifyTokenDetailed() in jwt.ts** — Reuses getSecret() logic and keeps token verification logic centralized
3. **Header Forwarding Pattern** — Uses Next.js 15 standard (immutable request) instead of trying to mutate request
4. **401 vs 403** — 401 for missing/invalid, 403 for expired (follows HTTP convention: 401 = auth required, 403 = auth failed)

## Security Analysis

✅ **Token Verification**
- Validates both signature and expiration
- Differentiates error types (401 vs 403)
- Returns null for errors, never throws (safe for middleware)

✅ **Cookie Handling**
- Extracts from httpOnly cookies (XSS protection)
- No plain-text tokens exposed
- Custom headers for internal request forwarding only

✅ **User Info Forwarding**
- Via custom headers (x-user-id, x-user-email)
- Internal to Next.js request pipeline (not exposed to clients)
- Safe pattern for server-side middleware
