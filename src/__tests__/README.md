# Tests Directory

Unit and integration tests for the application.

## Structure

- **components/** - Component tests
- **api/** - API route tests
- **lib/** - Utility and helper function tests
- **integration/** - Integration tests spanning multiple modules

## Conventions

- Test file names mirror source file names with `.test.ts` or `.test.tsx` suffix
- Tests colocated next to source files where appropriate, or organized here
- Use Jest and React Testing Library
- Follow TDD practices: write tests first
